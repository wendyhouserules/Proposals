CURSOR BRIEF — SailScanner Proposal Engine (Proposals + Proposal Yacht CPTs only)

SCOPE OF THIS PLUGIN
- This plugin implements ONLY: ss_proposal CPT, ss_proposal_yacht CPT, tokenized proposal URLs, and proposal/proposal-yacht templates.
- The yacht_charter CPT is OUT OF SCOPE. It is managed elsewhere (e.g. location/SEO pages) and must remain indexed. Do not register, noindex, or provide templates for yacht_charter. The files in examples/ are reference only for how external endpoints work.

CONTEXT
- SailScanner parses MMK/NauSYS email HTML and generates premium branded proposal pages on sailscanner.ai.
- Target UX: portal-style proposal page with sidebar on the LEFT, fixed (sticky) on desktop; on mobile sidebar at top or collapsible. Include a "How it works" section with steps.

PROPOSAL LAYOUT (single-ss-proposal.php)
- Sidebar: on the LEFT. Fixed/sticky on desktop; on mobile show at top (or collapsible at top). Contents: requirements, jump links, yacht picks list, contact, next steps.
- Main column (right): intro, yacht cards, itinerary, notes.
- "How it works" section: include a clear step-by-step section explaining the process (e.g. 1. Review your options 2. Shortlist & get in touch 3. We confirm your charter). Place in sidebar or main column as fits the design.

OBJECTIVES (v1)
1) Generate a tokenized proposal URL (/proposals/{token}/).
2) Store yacht picks as ss_proposal_yacht posts so "View details" has a real URL.
3) Render proposal template (intro, yacht cards, itinerary, notes) and ss_proposal_yacht single template (noindex).
4) Proposal and ss_proposal_yacht pages must be NOINDEX/NOFOLLOW and excluded from sitemap/search.

WORDPRESS DATA MODEL (THIS PLUGIN ONLY)
CPT: ss_proposal
- Token routing: /proposals/{token}/ resolves by meta ss_token.
- Meta: ss_token, ss_yacht_ids (ss_proposal_yacht post IDs), ss_yacht_data (JSON), ss_intro_html, ss_itinerary_html, ss_notes_html, ss_contact_whatsapp, ss_contact_email, ss_requirements_json, ss_status, ss_created_at, ss_expires_at.

CPT: ss_proposal_yacht
- One post per yacht pick in a proposal. Slug: proposal-yacht. Used only for "View details" URLs; no archive/search.
- Meta: display_name, images_json, highlights_json, cabins, berths, length_m, year, base_name, country, region, model, specs_json.

REST
- wp/v2: POST /ss_proposal with ss_yacht_data; server creates ss_proposal_yacht posts and sets ss_yacht_ids.
- Optional sailscanner/v1: POST /proposal with ss_yacht_data (HMAC + Basic auth).

TEMPLATES
- single-ss-proposal.php: portal layout with LEFT sidebar (fixed/sticky on desktop; at top on mobile), main content right; include "How it works" section with process steps.
- single-ss-proposal-yacht.php (yacht detail page, noindex).

NO-LEAKAGE
- Never display provider/operator names or MMK/NauSYS URLs on front-end.

DELIVERABLES
- ss_proposal + ss_proposal_yacht CPTs; token routing; proposal and proposal-yacht templates; REST for proposal create with ss_yacht_data; Python importer that POSTs ss_proposal with ss_yacht_data.
